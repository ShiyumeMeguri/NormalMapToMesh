# Normal Map To Mesh (法线 → 多级精度高模)

把切线空间法线贴图重建为 Multires(多级精度)修改器上的真实高模几何细节。
v2 全面改用**烘焙位移法**: Cycles 物体空间法线烘焙 + Displace(RGB→XYZ) 同款向量位移,
一键全自动, 可随时调强度重建。

## 用法

1. 选中目标网格(需有 UV)。`3D视图 > 侧栏(N) > Ruri > 法线 → 多级精度高模`。
2. **游戏资产**(材质里已有法线贴图节点链): 来源保持"自动", 直接点 **应用 / 更新**。
3. **裸网格 + 贴图文件**: 点 **加载法线并一键构建** 选文件(自动切到贴图模式)。
4. 调 **强度**(默认 0.02, 物体空间单位)后再点应用——从基面幂等重建, 不叠加。
5. **移除细节** 一键恢复低模。

## 原理(为什么 v1 失败, v2 正确)

- **v1(已废弃)** 用频域泊松积分把整张法线贴图当单一高度场在 UV 平面全局积分,
  隐含"UV 轴与切线基全局一致"的假设。平面/单岛测试成立; 真实游戏资产
  (实测发型: **406 个 UV 岛 / 144 个镜像岛 / 逐岛任意朝向的切线帧 / 混合手性**)
  上梯度方向逐岛错乱——这是前提性失败, 加多少岛间修正都救不了。
- **v2** 不再自己重建切线基: 在物体**真实材质**上用 Cycles 烘焙**物体空间法线**
  (Bake Type=Normal, Space=Object)。mikktspace 切线、绿通道 GL/DX 约定、
  游戏材质常见的通道重建节点网络、镜像/重叠 UV 岛, 全部由渲染器逐像素求值, 天然正确。
- 位移 = (烘焙RGB − 0.5) × 强度, 即 **Displace 修改器 RGB→XYZ** 的向量位移:
  幅值恒为 强度/2, 起伏来自逐像素方向差("法线转位移"的目视等效核心)。
  双面卡片正背面共享 texel 时得到同一向量、同向移动, 薄片不撕裂——
  v1 需要专门 BVH 背面识别才能近似做到的事在这里是免费的。
- 细分默认 **Simple**(保形); 位移经 `multires_reshape` 写入 Multires 位移层,
  可继续雕刻/再烘焙。
- 逐 loop 双线性采样 + 逐顶点平均(接缝顶点取两侧均值), 全 numpy 向量化。
- 烘焙结果按(网格指纹 × 材质 × 来源 × 分辨率)会话内缓存, 调强度重跑近乎零成本。

## 选项

| 选项 | 说明 |
| --- | --- |
| 来源 | 自动(材质优先)/ 材质(烘焙现有材质法线输出)/ 贴图(临时节点链) |
| 强度 | 物体空间单位, 语义 = Displace 修改器 Strength; 负值反向 |
| 烘焙分辨率 | 512~4096, 通常取源法线贴图同档(默认 2048) |
| 细分方式 | Simple(保形, 推荐)/ Catmull-Clark / Linear |
| 自动级别 / 四边形预算 | 级别自动匹配烘焙 texel 密度, 预算兜底内存 |

## 注意

- 烘焙期间会**临时**: 切渲染引擎为 Cycles(CPU)、隐藏其它物体、关掉本物体全部修改器的
  渲染求值(烘静息态基面法线)、往材质插入烘焙目标节点——结束后全部还原, 不留痕迹。
- 链接库材质无法插入烘焙节点, 请先 Make Local。
- 贴图模式会把文件图色彩空间自动置为 Non-Color(生成型图像跳过, 防清像素)。
- 物体上已有**非本工具创建**且带层级的 Multires 时会拒绝执行(防细节丢失)。

## 开发

`core.py` 为纯 numpy 数学内核, 不依赖 bpy:

```bash
python core.py   # 自测: 双线性采样 / RGB→XYZ 位移 / 接缝顶点平均
```
